1
Create a free Supabase project
Go to supabase.com → Sign up → New Project. Give it a name, set a strong database password, and pick the region closest to you. It takes about 60 seconds to provision.
2
Grab your API keys
In the Supabase sidebar go to Project Settings → API. You need two things:
• Project URL — looks like https://abcdef.supabase.co
• anon / public key — a long string starting with eyJ…
3
Paste the keys into this file
Open newsletter.html in any text editor. Near the top of the <script> block, find and fill in:
var SUPABASE_URL = 'https://abcdef.supabase.co'; var SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsIn...';
4
Create all database tables
In Supabase: go to SQL Editor → New Query. Paste the entire block below and click Run (▶).
-- 1. NEWSLETTERS CREATE TABLE newsletters ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, title text NOT NULL, author text, category text, kicker text, body text, created_at timestamptz DEFAULT now() ); ALTER TABLE newsletters ENABLE ROW LEVEL SECURITY; CREATE POLICY "Public read" ON newsletters FOR SELECT USING (true); CREATE POLICY "Anon insert" ON newsletters FOR INSERT WITH CHECK (true); CREATE POLICY "Anon delete" ON newsletters FOR DELETE USING (true); -- 2. CATEGORIES (custom only — 4 core are hardcoded) CREATE TABLE categories ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, name text NOT NULL UNIQUE, created_at timestamptz DEFAULT now() ); ALTER TABLE categories ENABLE ROW LEVEL SECURITY; CREATE POLICY "Public read" ON categories FOR SELECT USING (true); CREATE POLICY "Anon insert" ON categories FOR INSERT WITH CHECK (true); CREATE POLICY "Anon delete" ON categories FOR DELETE USING (true); -- 3. ACCESS CODES (6-char monthly rotating) CREATE TABLE access_codes ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, code text NOT NULL, month_key text NOT NULL UNIQUE, created_at timestamptz DEFAULT now() ); ALTER TABLE access_codes ENABLE ROW LEVEL SECURITY; CREATE POLICY "Anon read" ON access_codes FOR SELECT USING (true); CREATE POLICY "Anon insert" ON access_codes FOR INSERT WITH CHECK (true); -- 4. REACTIONS (per newsletter, per emoji) CREATE TABLE reactions ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, newsletter_id uuid NOT NULL REFERENCES newsletters(id) ON DELETE CASCADE, emoji text NOT NULL, session_id text NOT NULL, created_at timestamptz DEFAULT now(), UNIQUE(newsletter_id, emoji, session_id) ); ALTER TABLE reactions ENABLE ROW LEVEL SECURITY; CREATE POLICY "Public read" ON reactions FOR SELECT USING (true); CREATE POLICY "Anon insert" ON reactions FOR INSERT WITH CHECK (true); CREATE POLICY "Anon delete" ON reactions FOR DELETE USING (true);
5
Enable Realtime on all tables
In Supabase: go to Database → Replication. Toggle ON for:
• newsletters  • categories  • access_codes  • reactions

Save changes. Live updates will now work instantly for all users.
6
Get your first monthly access code
After saving your keys, the app auto-generates this month's 6-character code and saves it to the access_codes table. View it above or in Supabase → Table Editor → access_codes. Share it with your team. It rotates automatically on the 1st of every month.
7
Host the file
Drag newsletter.html to app.netlify.com/drop for instant free hosting. Or push to GitHub and enable GitHub Pages. Done in under 2 minutes either way.
